# Qwen3.5-2B text inference support

## Problem and scope
Add native text generation for the dense language model inside the official `Qwen/Qwen3.5-2B` checkpoint while preserving existing Qwen3 behavior. The distributed artifact is `Qwen3_5ForConditionalGeneration` and includes vision and MTP weights; export must consume only `model.language_model.*`, allowlist-skip `model.visual.*` and `mtp.*`, and reject unknown text tensors/configurations. Vision inputs and MoE are unsupported.

## Pinned local reference
- Model artifact: `sample/Qwen/Qwen3.5-2B/`.
- Normative implementation: `sample/.venv/Lib/site-packages/transformers/models/qwen3_5/{configuration_qwen3_5.py,modeling_qwen3_5.py}`.
- Config reports Transformers `4.57.0.dev0`; immutable external revision hashes are unavailable, so local files are the reproducibility reference for this change.

## Success criteria
- Export the supplied official wrapper without loading vision/MTP tensors and without missing, duplicating, or silently misordering any language tensor.
- Run correct text-only autoregressive generation with hybrid caches, explicit reset, and Qwen3.5 chat termination.
- Preserve Qwen3 v1 checkpoint compatibility.
- Pass workspace formatting, Clippy, and tests; compare deterministic primitive/cache outputs against fixtures generated by the pinned local Transformers source.

## Exact architecture
Text config: vocab 248320, hidden 2048, FFN 6144, 24 layers, 8 query heads, 2 KV heads, full-attention head dim 256, max positions 262144, epsilon 1e-6. Layers 3,7,11,15,19,23 are full attention; the other 18 are Gated DeltaNet.

Standard decoder/final/QK norms use `rms(x) * (1 + weight)`. DeltaNet gated norm uniquely uses `rms(x) * weight * silu(z)`. These conventions must remain distinct.

Full attention uses q projection output width `2 * n_heads * head_dim`; split each head into Q and output gate, apply Q/K norm and partial RoPE to the first 64 of 256 dimensions with theta 10,000,000, perform GQA causal attention, multiply concatenated output by `sigmoid(gate)`, then apply o projection. For text-only positions all three MRoPE axes equal the token position, so interleaving collapses to ordinary partial RoPE.

Each DeltaNet layer projects QKV (6144), z (2048), a (16), and b (16), applies depthwise causal convolution of width 4 and SiLU, then splits 16 heads of Q/K/V at 128 dimensions. Q/K use L2 normalization; Q is scaled by `1/sqrt(128)`. `beta=sigmoid(b)` and `g=-exp(A_log)*softplus(a+dt_bias)` in FP32. Per token and head: decay state by `exp(g)`; compute `delta=(v-S^T k)*beta`; update `S += k outer delta`; output `S^T q`. Apply the distinct gated norm and out projection. Cache per linear layer is FP32 convolution state `[6144,4]` plus recurrent state `[16,128,128]`.

MLP is SwiGLU `down(silu(gate(x))*up(x))`. Embeddings and LM head are tied.

## Checkpoint v2
- Keep Qwen3 as `(version=1, architecture_id=1)`. ID 2 remains reserved for Llama. Qwen3.5 uses architecture ID 3 and version 2.
- Retain the 256-byte header. V2 stores fixed-width little-endian fields: common dimensions/group/shared-classifier, norm epsilon, RoPE theta and rotary dimension, linear head counts/dimensions, convolution width, a 64-bit full-attention layer mask, model EOS, and chat EOS. Validate positive dimensions, divisibility, supported activation/bias/gating/tie settings, schedule length/mask, and checked byte arithmetic.
- Tensor stream order: standard norms (input/post for all layers, final, full-layer Q/K norms); Delta gated norms; FP32 Delta `A_log`, `dt_bias`, and depthwise conv weights; quantized tied embedding; then per layer in layer order: mixer tensors in exact architecture-specific order followed by gate/down/up MLP; no LM head when tied. Exporter and reader share documented tests for this order.
- Standard norm weights remain raw and runtime applies `1+w`; Delta gated norm remains direct. `A_log`, `dt_bias`, and convolution weights stay FP32 to avoid tiny-tensor group-size failures and recurrent instability. Large linear matrices use existing Q8 format.

## Runtime and cache lifecycle
- Add architecture dispatch and an explicit `reset_cache()` Transformer operation. Chat context rollover invokes it; position zero also asserts/establishes fresh state.
- Add a prompt-prefill API. Initial implementation may execute the exact recurrent rule token-by-token but computes the 248320-way LM head only for the final prompt token. Chunk-size-64 DeltaNet prefill is a later optimization, not an MVP correctness requirement.
- Qwen3.5 defaults to a 4096-token runtime context when `--context` is omitted; explicit values are bounded by the model maximum. At 4096, six FP32 full-attention K/V caches are about 96 MiB; linear recurrent+conv state is about 20 MiB. Checked allocation failures return errors. The 262144 model maximum is metadata, not the default allocation (full-attention cache alone is ~6 GiB).
- Qwen3.5 embedding lookup dequantizes only the selected row, avoiding a ~1.9 GiB FP32 embedding copy.

## Tokenizer/chat
The model config EOS is 248044 (`<|endoftext|>`), while tokenizer chat EOS is 248046 (`<|im_end|>`). Export tokenizer EOS from `tokenizer_config.json`, so chat/raw CLI generation terminates on 248046; retain model EOS in v2 metadata as an additional stop token. Text chat formatting is `<|im_start|>{role}\n...<|im_end|>\n`; generation begins `<|im_start|>assistant\n` followed by either `<think>\n` or `<think>\n\n</think>\n\n`. Image/video content is not exposed by the CLI.

## Tests and acceptance
- Config fixture: nested text config, exact layer mask, strict unsupported-field errors, official-wrapper acceptance, MoE/unknown text rejection.
- Export: real tensor names/shapes, allowlisted visual/MTP skips, every language tensor consumed exactly once, tied head, stream order, truncated/malformed v2 rejection, v1 regression.
- Primitive fixtures: standard and gated RMSNorm, partial RoPE, gated full attention, convolution update, L2 norm, FP32 `g/beta`, recurrent Delta state/output at lengths 1–4 and reset. If chunk prefill is later added, test boundaries 63–65.
- Runtime: prompt prefill versus repeated decode equivalence, final-only logits during prefill, cache reset/rollover, 4096 default context, checked maximum override.
- End-to-end: tokenization/chat string equality and greedy logits/tokens against the pinned local Transformers model. Separate quantization-emulated references from BF16 to avoid conflating Q8 error with kernel error.
- Commands: `cargo fmt --all -- --check`; `cargo clippy --workspace --all-targets -- -D warnings`; `cargo test --workspace`; release export and short greedy generation from the supplied model.

## Alternatives rejected
- Alias to Qwen3: equations, norms, q projection, layer schedule, and cache differ.
- Reject all vision tensors: the official target wrapper contains them; only allowlisted skipping makes the requested artifact exportable.
- Include vision/MoE: outside scope.
- Require chunked prefill before correctness: adds substantial complexity; final-only LM-head prefill addresses the largest avoidable per-token cost while preserving an exact simple recurrence.

## Implementation status and residual risks
The v2 exporter, hybrid single-token inference, cache lifecycle, generation integration, tokenizer EOS handling, and documentation are implemented. Workspace formatting, checking, warning-free Clippy, 66 tests/doctests, a real 1.9 GiB Q8 export, and an eight-token-context greedy smoke generation all pass for the supplied model. Numerical parity acceptance remains incomplete.

- A Transformers BF16 comparison could not run because the supplied virtual environment has Transformers sources but no `torch` installation.
- Strict inventory/shape validation for every `model.language_model.*` safetensor is not yet complete; required tensors fail when missing, but unknown text tensors are not exhaustively rejected.
- Scalar Rust Q8 kernels and per-token temporary allocations may make CPU generation slow; release benchmarks on the user's hardware must be reported rather than claiming throughput.
- Q8 recurrent error can accumulate. Primitive and end-to-end reference evidence is mandatory before production use.
- The local dev Transformers source lacks an immutable package revision; file hashes should be captured when command execution is available.
